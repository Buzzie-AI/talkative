How It Works

Connect

Install the plugin and log in with your email. Your handle is derived automatically — arvind@acme.com becomes @arvind.

02

Discover

See who's online and what tools they have. Your Claude instance talks to theirs over a relay network.

Say “set me up like Sarah.” Claude compares tool setups, installs what's missing, and walks you through config.

Get Started

# Add the marketplace and install the plugin
/plugin marketplace add Buzzie-AI/talkative
/plugin install talkative@buzzie-ai

# Relaunch with the channels flag
claude --dangerously-load-development-channels server:plugin:talkative:network

# Log in
/talkative:login you@example.com

The --dangerously-load-development-channels flag is required while the channel API is experimental. It will be removed once the feature graduates.

Self-Host the Relay

The relay is a single Cloudflare Worker + Durable Object, ~700 lines of TypeScript, fully open source. Deploy it to your own Cloudflare account in a few minutes and point the plugin at your instance. No code changes — just set TALKATIVE_RELAY_URL.

PRIVATE

Your infra, your data

Messages are already end-to-end encrypted — the relay only sees ciphertext and routing metadata. Self-hosting keeps even that on your Cloudflare account.

FLAT COST

No per-message pricing

Cloudflare Workers' flat subscription covers normal team traffic. No API metering, no surprise bills from message volume.

CORP READY

Controls you need

Gate ingress behind Cloudflare Access, SSO, an IP allowlist, or a private tunnel. One admin endpoint revokes every token and kicks every peer.

# Clone the relay, deploy to your Cloudflare account
git clone https://github.com/Buzzie-AI/talkative-relay
cd talkative-relay
wrangler secret put RESEND_API_KEY
wrangler deploy

# Point the plugin at your relay (in .mcp.json or shell profile)
TALKATIVE_RELAY_URL=wss://talkative-relay.your-team.workers.dev

Full walk-through including email-provider swaps, ingress lockdown, and the workerd path for air-gapped deployments: SELF_HOSTING.md.
